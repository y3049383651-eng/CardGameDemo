# 程序设计说明

## 1. 目标

本项目当前实现的是文档中“二、任务实现需求”对应的关卡版本，重点完成了以下能力：

- 按关卡配置文件生成主牌区与手牌区卡牌
- 支持手牌区替换顶部牌
- 支持主牌区卡牌按 `+1/-1` 规则匹配顶部牌
- 支持撤销，卡牌可回到原位置并恢复上一张顶部牌

按 `configs / models / views / controllers / managers / services` 分层组织。

## 3. 运行流程

### 3.1 游戏启动流程

1. `HelloWorldScene::init()` 创建 `GameView`
2. `HelloWorldScene::init()` 创建 `GameController`
3. `GameController::startGame()` 调用 `LevelConfigLoader` 读取 JSON
4. `GameController::startGame()` 调用 `GameModelFromLevelGenerator` 生成 `GameModel`
5. `GameController::createCardViews()` 通过 `CardController` 创建主牌区和手牌区卡牌视图
6. `GameController::refreshTopCard()` 创建顶部牌视图

### 3.2 点击卡牌流程

1. `CardView` 捕获点击
2. `CardView` 调用注册好的回调
3. `CardController` 将回调按卡牌 `id` 转发回 `GameController`
4. `GameController::handleCardSelection()` 执行规则判断
5. `GameController` 在真正变更前写入 `UndoManager`
6. `CardController::moveCardTo()` 执行动画
7. `GameController::finishCardMove()` 更新 `GameModel`
8. `GameController::refreshTopCard()` 刷新顶部牌显示

### 3.3 撤销流程

1. 用户点击 Undo 按钮
2. `HelloWorldScene` 调用 `GameController::undo()`
3. `GameController` 从 `UndoManager` 弹出最近记录
4. `GameController` 恢复顶部牌状态
5. `CardController::restoreCard()` 播放回位动画
6. 动画结束后重新绑定点击回调

## 4. 关键设计点

### 4.1 为什么配置和运行时模型分开

`LevelConfig` 表示“关卡原始描述”，适合放在资源文件里。  
`GameModel` 表示“当前对局状态”，会随着点击和撤销不断变化。  
这两层分开后：

- 关卡设计更容易替换
- 运行时逻辑不会污染静态配置
- 以后新增多个关卡时，只要新增 JSON 文件即可

### 4.2 为什么用 CardController

`GameController` 负责规则和流程，`CardController` 负责具体卡牌视图操作。  
这样做的好处是：

- `GameController` 不需要直接管理大量 `CardView*`
- 动画逻辑集中，后续改动画更方便
- 回调绑定和恢复只需要维护一份

## 5. 如何新增一个卡牌

只需要改配置文件。

做法：

1. 打开 [task2_level.json](/D:/projects/CardGame/Resources/configs/levels/task2_level.json)
2. 在 `Playfield` 或 `Stack` 数组中追加一项
3. 填入 `CardFace`、`CardSuit`、`Position`
4. 重新运行游戏

例如，新增一张黑桃 5 到主牌区：

```json
{
  "CardFace": 4,
  "CardSuit": 3,
  "Position": { "x": 500, "y": 900 }
}
```

## 6. 总结

当前这套结构已经满足文档要求中的核心分层思想：

- `configs` 管静态数据
- `models` 管运行时状态
- `views` 管显示和输入
- `controllers` 管流程和用户操作
- `managers` 管撤销记录
- `services` 管无状态转换和规则辅助

在这个基础上，后续新增关卡、普通卡牌、特殊卡牌、或者新类型撤销功能，都可以沿着现有层次继续扩展，而不需要把所有逻辑重新塞回 `GameController`。
